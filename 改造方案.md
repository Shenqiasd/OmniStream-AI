### 3. 移除订阅验证（重要）

**订阅验证位置**：
- VIP 服务：`/project/backend/apps/aitoearn-server/src/user/vip.service.ts`
- 用户服务：`/project/backend/apps/aitoearn-server/src/user/user.service.ts`
- AI 服务：`/project/backend/apps/aitoearn-server/src/ai/` 目录下的各个服务

**移除方法**：

#### 方法一：修改 VIP 检查逻辑（推荐）

找到所有调用 VIP 检查的地方，注释掉或返回 true。

**需要修改的文件**：
```bash
# 搜索所有 VIP 检查的地方
grep -r "checkVip\|isVip\|vipLevel" /project/backend/apps/aitoearn-server/src --include="*.ts"
```

**示例修改**：
```typescript
// 原代码
async checkVipAccess(userId: string) {
  const user = await this.userService.findById(userId)
  if (!user.isVip) {
    throw new AppException(ResponseCode.VipRequired)
  }
}

// 修改后（直接返回 true，跳过检查）
async checkVipAccess(userId: string) {
  return true  // 移除 VIP 限制
}
```

#### 方法二：修改用户默认 VIP 状态

修改用户创建时的默认 VIP 状态为 true。

**文件**：`/project/backend/libs/mongodb/src/schemas/user.schema.ts`

```typescript
// 找到 isVip 字段，修改默认值
isVip: { type: Boolean, default: true }  // 改为 true
```

### 4. 配置平台 OAuth（添加频道必须）

**问题**：添加频道报错是因为缺少平台 OAuth 配置。

**配置文件**：`/project/backend/.env`

**需要配置的平台**：

```bash
# TikTok
TIKTOK_CLIENT_ID=your-tiktok-client-id
TIKTOK_CLIENT_SECRET=your-tiktok-client-secret
TIKTOK_REDIRECT_URI=http://localhost:3002/callback/tiktok

# Twitter/X
TWITTER_CLIENT_ID=your-twitter-client-id
TWITTER_CLIENT_SECRET=your-twitter-client-secret
TWITTER_REDIRECT_URI=http://localhost:3002/callback/twitter

# YouTube
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_REDIRECT_URI=http://localhost:3002/callback/youtube

# Meta (Facebook/Instagram)
META_FACEBOOK_CLIENT_ID=your-facebook-app-id
META_FACEBOOK_CLIENT_SECRET=your-facebook-app-secret
META_FACEBOOK_REDIRECT_URI=http://localhost:3002/callback/facebook
```

**获取 OAuth 凭证**：
1. **TikTok**: https://developers.tiktok.com/
2. **Twitter**: https://developer.twitter.com/
3. **YouTube**: https://console.cloud.google.com/
4. **Meta**: https://developers.facebook.com/

## 常见问题排查

### 问题 1：发布功能仍然报错

**可能原因**：
1. 后端服务未重启
2. 数据库连接失败
3. 缺少必要的环境变量

**解决方法**：
```bash
# 检查服务是否运行
ps aux | grep "aitoearn"

# 查看服务日志
tail -f /tmp/aitoearn-*.log

# 检查数据库连接
mongo --host localhost --port 27017 -u admin -p change-this-mongodb-password
```

### 问题 2：添加频道失败

**可能原因**：
1. 缺少平台 OAuth 配置
2. 回调地址不正确
3. OAuth 应用未审核通过

**解决方法**：
1. 确认已配置对应平台的 OAuth 凭证
2. 检查回调地址是否与平台配置一致
3. 在平台开发者控制台检查应用状态

### 问题 3：AI 生成失败

**可能原因**：
1. AI API Key 未配置或无效
2. API 配额不足
3. 网络连接问题

**解决方法**：
```bash
# 测试 OpenAI API
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"

# 检查环境变量
echo $OPENAI_API_KEY
```
